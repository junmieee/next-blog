- parcel 패키지를 설치해보자
    
    Parcel은 lodash같은 여러가지 패키지들을 웹사이트에 올라갈 수 있는 하나의 구조로 묶어주는 역할을 한다. 따라서 이 패키지는 ‘번들러(Bundler)라고 부른다.
    
    이 패키지는 웹사이트에서 동작하는 내용들을 하나로 묶어주는 역할을 하는 것이지, 실제로 웹사이트에서 동작하는 것은 아니다(자바스크립트 코드는 실제 컴퓨터를 제어하는 역할로도 사용하고 브라우저에서 웹사이트를 동작시키는 역할로도 사용한다.) 따라서 패키지 설치 명령어는 다음과 같다. 
    
    ```jsx
    npm install parcel --save-dev
    ```
    
    → parcel이라는 패키지를 저장해서 사용할 건데, 개발할 때만 사용하고 실제 웹브라우저에서 동작하는 패키지는 아니다. 
    
    → 따라서 패지를 설치할 시 개발용으로만 사용할 것인지 lodash처럼 실제 브라우저에서 동작하는 기능을 담고 있는지 구분할 줄 알아야한다. 
    
    *아래 명령어는 약어이다.
    
    ```jsx
    npm install -D parcel
    ```
    
    설치를 완료하면 아래와 같이 devDependencies에 parcel 패키지가 설치된 것을 볼 수 있다. 
    
    ![스크린샷 2023-01-12 오후 4.31.06.png](https://s3-us-west-2.amazonaws.com/secure.notion-static.com/138040d0-5982-4490-a969-65c4bc3b89e4/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2023-01-12_%E1%84%8B%E1%85%A9%E1%84%92%E1%85%AE_4.31.06.png)
    
    *package.json : 나의 프로젝트가 직접적으로 의존하고 있는 패키지를 관리하는 파일 
    
    *package-lock.json: 설치한 패키지들에 연결이 되어져 있는 다양한 패키지들의 관계나 버전이 명시되어 있음
    
    node_modules 파일을 삭제하더라도 아래의 명령어로 언제든지 원상복구 할 수 있다(그 이유는 우리가 가지고 있는 package-lock.json과 packagge.json파일이 있기 때문)
    
    ```jsx
    npm install
    ```
